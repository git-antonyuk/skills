name: learning-worked-example
description: Teach a topic using the Worked Examples method. Use when the user wants to learn and retain a concept — "teach me", "explain step by step", "walk me through", "help me understand the pattern behind X" — rather than just get a finished answer.
---

Don't give the final solution or raw code. Teach the pattern using a worked example.

Restate the problem in one line, then break the solution into minimal, atomic, chronological steps. For each step, explain the underlying logic and give the single concrete move that solves it. Keep one idea per step so cognitive load stays low.

After the breakdown, summarize the reusable pattern so it transfers to similar problems.

As the user gains competence, fade the support: skip steps or leave gaps for them to fill in. Start with full examples for new concepts.
